Quelle est la problématique ?
-----------------------------

La prospection commerciale peut vite devenir coûteuse et chronophage. Les abonnements aux bases de données de prospects sont souvent onéreux, et la constitution manuelle de fichiers est interminable.

L'objectif est ici de pouvoir extraire et enrichir une base de données clients en moins d'une heure, en ciblant n'importe quel prospect en France, sans abonnement fixe, mais en payant uniquement à la donnée extraite (pay-as-you-go).

### La Stack Technique

Ce système repose sur l'orchestration de plusieurs outils performants et économiques :

*   **n8n** : Pour l'automatisation des workflows (alternative plus flexible à Make ou Zapier).
*   **Outscraper** : Une API puissante pour scraper des données depuis Google Maps, LinkedIn, etc.
*   **NocoDB** : Pour stocker et organiser les données (base de données NoCode).
*   **Brevo** : Pour la gestion des campagnes d'emailing.

### Les Étapes du Workflow

Le processus commence par définir une liste de villes cibles (ex: Paris, Marseille, Lyon) et une requête spécifique (ex: "Hôtel").

Le workflow boucle sur ces paramètres pour interroger Google Maps via Outscraper. Cela permet de récupérer les fiches entreprises correspondant à la recherche dans les zones géographiques définies.

#### 2. Enrichissement et Vérification des Données

Une fois les entreprises identifiées, le système procède à plusieurs vérifications cruciales pour qualifier la donnée :

*   **Vérification du domaine** : On s'assure que l'entreprise possède un site web et un nom de domaine valide.
*   **Scraping de contacts** : Extraction des adresses emails et numéros de téléphone associés.
*   **Validation des emails** : Une étape indispensable pour garantir la délivrabilité. Le système vérifie si l'email est valide pour éviter les "bounces" qui pourraient nuire à la réputation de votre domaine d'envoi.

Si aucune adresse email valide n'est trouvée, le prospect est écarté pour ne pas polluer la base de données.
